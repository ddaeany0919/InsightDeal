# InsightDeal 마스터 기획 및 계획표 (Plan)

본 문서는 InsightDeal의 개발 현황, 로드맵, UI/UX 가이드라인을 하나로 통합한 유일한 기획 문서입니다.

---

## 🎨 UI/UX 톤앤매너 매뉴얼 (Brand Voice)
- **핵심 철학**: "심플 이즈 베스트!" 조작은 직관적이어야 하며, 복잡한 사용법을 요구해서는 안 됨.
- **애니메이션 원칙**: 아이템 추가/삭제 시 끊김 없이 쫀득한(`spring` 스펙) Compose 트랜지션 애니메이션 적용.
- **상태 처리**: 로딩 중이거나 텅 빈 화면일 때 심심한 텍스트 대신 `Lottie`나 일러스트를 배치하여 빈 상태(Empty State) 디자인 보장.
- **문구 톤**: 딱딱한 시스템 메시지는 지양하고 "현재 가격을 열심히 불러오고 있어요 🚀" 같이 모던 앱의 친근한 문체 사용.

## ✅ 완료된 작업 (Completed)

### 1. 안드로이드 앱 (Frontend)
- **최적의 5-Tab 네비게이션 구조 확립**: [홈] - [검색] - [카테고리] - [커뮤니티] - [내정보] 순으로 상용 커머스 앱 수준의 UX 동선 최적화 완료.
- **화면 기능 분리 및 재배치**: 
    - '관심목록(찜)'과 '설정'을 [내정보] 탭 내부로 통합하여 하단 바 간소화.
    - '키워드 알림' 센터를 홈 화면 최상단(Top App Bar)의 종(🔔) 아이콘으로 이동.
    - 홈 화면 최상단 카테고리 칩셋은 '퀵 필터'로, 하단 탭의 카테고리는 '풀 디렉토리' 역할로 이원화.
- **데이터 렌더링 최적화 및 AI UI 고도화 [수정/신규]**: 
    - 홈 화면의 꿀딜 추천도 아이콘(🍯)을 덜어내고, 프로페셔널한 AI 분석 지표("💡 AI 가격 메리트") 텍스트로 리팩토링.
    - 초특가 배지 조건 강화 (85점 이상 + 역대가 키워드).
    - UI 텍스트 "무료"로 간소화 및 '이벤트' 정상 노출.
    - 상세 페이지(`DealDetailScreen`)에서 가격 히스토리 데이터가 2개 이하로 부족할 때, 차트 영역 빈 공간을 친화적인 안내 메시지로 채우고, AI 구매 가이드에 카테고리 평균 대비 추천도를 폴백(Fallback)으로 제공하도록 UI 고도화 완료.
- **리팩토링**: Material 3 디자인 가이드에 맞춘 카드 UI(RoundedCorner, Elevation) 일괄 적용 및 Deprecated 코드 제거.

### 2. 백엔드 (Backend)
- **멀티 커뮤니티 스크래퍼 통합 및 안정화**: 뽐뿌, 펨코, 퀘이사존, 클리앙 등 9개 커뮤니티 비동기 파이프라인 완벽 가동. 펨코 사이드바 가격(23,200원) 오인식 버그 완전 해결 및 본문/이미지/구매링크 추출 100% 정상화.
- **카테고리 정밀 분류 시스템 구축**: '적립' 카테고리를 100원 이하로 강제 할당하여 고가 핫딜 오분류 원천 차단. '드라이브' 등 오인식 키워드를 예외 처리하여 '이벤트' 분류 정확도 극대화.
- **자동 품절/종료 감지 시스템(Ping Daemon)**: 백그라운드 스케줄러가 과거 3일 내 핫딜의 아웃링크 404 및 제목 마감 상태를 감지하여 `is_closed` 상태를 자동 업데이트하는 데몬 가동 완료.
- **Two-Track 오케스트레이션 엔진**: 5분 주기 1페이지 단기 스캔과 1시간 주기 3페이지 심층 스캔을 교차로 수행하는 `update_all_deals.py` 및 스케줄러 구조 확립.
- **데이터베이스 무결성 확보**: 모든 휴리스틱 가격 오기입을 클렌징하고 `is_closed` 및 카테고리를 소급 적용하는 일괄 갱신(`reprocess_deals.py`) 완료.
- **꿀딜(Honey Score) 알고리즘 DB 연동화 [수정/신규]**: 기존 휴리스틱 난수 방식에서 벗어나, DB에 적재된 동일 카테고리 평균 가격과 현재 할인가를 비교 분석하여 더 정교한 가격 메리트 점수를 산출하도록 개선 완료.
- **핫딜 전용 실시간 트렌드 수집 [수정/신규]**: 펨코 핫딜 게시판의 실시간 급상승 키워드를 직접 스크래핑하여 캐싱하도록 변경 (기존 커뮤니티 전체 트렌드 -> 핫딜 특화 키워드로 변경).
- **푸시 알림 매칭 및 수익화 엔진 완공**: `AggregatorService`에 키워드 매칭, 일 3회 Rate Limiting(피로도 방지), 정보통신망법(야간 21시~08시) 발송 차단, 쿠팡/알리익스프레스 수익화 링크 우회 로직 구현 완료 및 테스트 검증.

### 3. 데이터 품질 및 UI 정확도 개선 (이번 세션 완료)
- **🔥 핫딜 오탐지(뱃지 오분류) 버그 완전 수정**: `HomeScreen.kt`의 `isSuperHot` 조건을 `honeyScore >= 95` 단독에서 **`🔥 태그 포함 AND honeyScore == 100` 이중 조건**으로 강화. 일반 게시글이 핫딜로 잘못 표시되던 문제 종식.
- **뽐뿌 `pmarket` 게시판 오염 제거**: 게시판 화이트리스트 필터(`ppomppu`, `ppomppu4`, `ppomppu8`만 허용)가 이미 수확 확인. `pmarket` 게시판에서 유입된 기존의 3건 DB 데이터 `🔥` 태그 삭제 및 honey_score 100점고선 소급 수정 완료.
- **커뮤니티별 `is_super_hotdeal` 판별 기준 체계화**: 뽐뿌(hot_icon2.jpg), 루리웹(best 클래스), 퀘이사존(인기 라벨/좋아요≥20), 펨코(포텐), 클리앙(공감≥10), 빠삭(조회≥2500) 등 각 커뮤니티별 시스템 내 기준 확인 및 문서화.
- **에펨코리아 스크래퍼 다중 본문 파싱 개선**: 다중 `.xe_content` 구조 특성상 첫 번째 블록만 파싱하던 버그를 수정하여 모든 본문 블록 병합 및 `<a>` 링크 보존 로직으로 고도화. AI 모음전 분할 시 누락 상품 방지.
- **모음전 Fallback 접두사 수정**: `aggregator_service.py`의 모음전 데이터 폴백텍스트에서 `🔥 [모음전]` → `📦 [모음전]`으로 변경, 이모지 기반 핫딜 뱃지 오발동 차단.

### 4. 커뮤니티 UGC 및 인증 시스템 (이번 세션 완료)
- **회원가입 및 인증(Auth) 아키텍처**: `User` 테이블 신설(아이디, 닉네임, 암호화 비밀번호, 꿀점수) 및 FastAPI `/api/auth` 라우터 구현 완료.
- **안드로이드 세션 관리**: DataStore(Preferences) 기반의 `AuthManager.kt`를 도입하여 로그인 유저의 닉네임과 세션을 영구적으로 로컬 캐싱.
- **핫딜 역경매 (찾아주세요)**: 유저가 직접 핫딜을 찾아달라고 요청하고, 도와준 유저에게 `bounty_points`(꿀점수)를 지급하는 게이미피케이션 커뮤니티 기능 백엔드 및 UI 연동 완료.
- **오프라인 핫딜 (발품지도)**: 크롤러가 잡지 못하는 동네 마트 마감세일 등 오프라인 전용 핫딜 제보 게시판(`post_type="OFFLINE"`) 아키텍처 확립.

### 5. 2단계 최종 고도화 (오프라인 캐싱, 보안 암호화 및 스마트 FCM 푸시)
- **오프라인 우선 (Offline-First) UX 아키텍처 완비**: 
    - `DealEntity` 및 `DealRemoteKeysEntity` 기반의 Room 데이터베이스를 추가 구축.
    - Paging 3 `HotDealsRemoteMediator`를 적용하여 API 호출 실패나 네트워크 두절 시에도 로컬 캐시 데이터를 최종 Single Source of Truth(단일 진실 공급원)로 삼아 끊김 없는 핫딜 스크롤 리스트 렌더링 지원.
    - 스크롤 시 빈번한 직렬화/역직렬화 부하로 인한 UI 프레임 드랍을 막기 위해 `Converters.kt` 내부 Gson을 static companion object 싱글톤으로 고정 바인딩하여 쫀득한 렌더링 성능 확보.
- **실시간 네트워크 상태 감지 배너**: 
    - `ConnectivityManager` 콜백을 코루틴 `callbackFlow`로 스트리밍하는 `NetworkMonitor` 설계.
    - 오프라인 단절 즉시 `HomeScreen` 최상단에 미려한 페이드 슬라이딩 애니메이션 경고 배너(오렌지 `#FFFF9500` 테마 컬러)를 띄워 기기 상태를 사용자에게 투명하게 공유.
- **하드웨어 수준의 로컬 보안 암호화 마이그레이션**:
    - `androidx.security:security-crypto:1.1.0-alpha06`을 결합하여 Android Keystore에 마스터 키를 암호학적으로 안전하게 저장.
    - 디바이스 잠금으로 백그라운드 스레드에서 Keystore 로드가 거부되는 안드로이드 특유의 런타임 예외를 극복하기 위해, 예외 감지 시 일반 SharedPreferences 암호화 백업 영역으로 즉각 우회하는 Fallback 싱글톤 빌더 아키텍처 구현.
    - 앱 잠금 PIN 관련 설정(`app_lock_enabled`, `app_lock_pin`), 야간 DND 정보(`night_push_consent`), 로컬 알림 이력 데이터(`NotificationHistoryManager`)의 스토리지를 일괄 보안 마이그레이션 완료.
- **스마트 FCM 푸시 및 야간 방해금지(DND) 필터링**:
    - `InsightDealFirebaseMessagingService` 백그라운드 리스너 완비 및 `AndroidManifest.xml` 안전 등록 완료. 백그라운드 푸시 수신 시 즉시 로컬 알림 보관함에 은밀히 자동 적재.
    - 푸시 수신 시간이 야간(21:00 ~ 08:00) 시간대일 경우 사용자의 DND 활성화 여부에 따라 동작을 스마트 분기. DND 활성화 상태 시 알림 소리/진동을 완전 무음 처리하여 로컬 보관함에 조용히 저장하며, DND 비활성화 상태에서는 무음 전용 채널(`silent_hotdeal_notifications`)로 분기 처리하여 알림 피로도 및 기기 이탈률 원천 차단.

### 2. 백엔드 (Backend)
- [ ] **대량 푸시 전송 비동기 워커 (Bulk Push Sender)**: 1만 명 이상의 유저에게 실시간 FCM 알림을 동시에 발송할 때 병목 현상 및 시간 지연을 차단하기 위한 비동기 멀티스레딩 셀러리(Celery) 큐 배치 구축.
- [ ] **Algolia 초고속 인덱싱 엔진**: 핫딜 게시판 데이터 적재 즉시 Algolia 검색 인덱스를 실시간 업데이트하여 프론트엔드 자동완성 및 0.1초 검색 구현.
- [ ] **제휴 마케팅 수수료 파트너스 ID 연동 및 동적 백엔드 어필리에이트 URL 생성 기능 구축**: 
    - 백엔드 `.env` 파일에 쿠팡 파트너스 ID(`COUPANG_PARTNERS_ID`) 및 알리익스프레스 어소시에이트 ID(`ALI_ASSOCIATES_ID`)를 환경변수로 관리.
    - 핫딜 수집 및 아웃링크 이탈 시, 백엔드 링크 제네레이터가 동적으로 어필리에이트 서브/파트너스 파라미터를 조립하여 프론트엔드에 전달함으로써 수익화 인프라 최적화.
    - **[추가 요구사항]** 안드로이드 클라이언트 측: 
        * 제휴 수수료 고지 BottomSheet UI 상에 각 플랫폼별(쿠팡/알리) 어필리에이트 안내 문구를 동적으로 출력.
        * 외부 이탈 링크가 쿠팡 또는 알리익스프레스 도메인일 경우에만 제휴 고지 BottomSheet가 안전하게 필터링되어 발동하도록 정밀 도메인 체크 함수 결합.
        * 백엔드에서 전달된 파트너스 연동 파라미터가 포함된 최종 URL로의 브라우저 아웃바운드 랜딩 보장.

### 2. 차세대 AI & 커뮤니티 엔진 (v2.0)

- [ ] **하이브리드 협업 필터링 추천 시스템**: 온디바이스 행동 패턴 분석과 클라우드 추천 알고리즘을 결합하여, 유저 취향에 100% 매칭되는 '타겟형 초정밀 꿀딜 피드' 제공.
- [ ] **동네 오프라인 핫딜 기반 지도 (LBS)**: 유저가 속한 GPS 위치를 기반으로 주변 식자재마트, 편의점 마감 할인 정보를 지도 UI상에 동적으로 매핑하여 지역 밀착형 하이퍼로컬 커머스 생태계 구축.
